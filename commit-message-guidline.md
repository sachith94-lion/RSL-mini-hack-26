# Commit Message Guidelines

We use the **Conventional Commits** standard to keep the history readable for judges:

### Format
`<type>: <description>`

### Types
- **feat**: A new feature (e.g., `feat: add firestore habit tracking`)
- **fix**: A bug fix (e.g., `fix: mobile padding on dashboard`)
- **docs**: Documentation changes only.
- **style**: Changes that do not affect the meaning of the code (CSS, formatting).
- **refactor**: Code change that neither fixes a bug nor adds a feature.

### Example
`feat: implement angular signals for real-time habit updates`
